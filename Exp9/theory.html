Three Address Code (TAC)

Three Address Code is a commonly used intermediate code representation in compilers. Each TAC instruction contains at most three addresses: two operands and one result.

Purpose of TAC

  • Simplifies complex expressions.
  • Facilitates code optimization.
  • Acts as an intermediate representation between source code and machine code.
  • Helps in target code generation.

General Form

x = y op z
where:
  • x → result
  • y, z → operands
  • op → operator

Example

Expression:
a + b * c

Three Address Code:

t1 = b * c
t2 = a + t1

Types of TAC Statements

  • Assignment Statements
  • Binary Operations
  • Unary Operations
  • Conditional Jumps
  • Unconditional Jumps
  • Procedure Calls

Applications

  • Intermediate Code Generation
  • Compiler Optimization
  • Code Translation
  • Machine Code Generation
